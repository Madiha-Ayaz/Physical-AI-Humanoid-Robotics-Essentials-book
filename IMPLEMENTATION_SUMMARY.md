### Content Quality Standards

1. **Length**: 800-2000 words per chapter
2. **Code Examples**: Include working Python code with syntax highlighting
3. **Admonitions**: Use `:::tip`, `:::warning`, `:::info` for callouts
4. **Tables**: Use markdown tables for comparisons
5. **Links**: Cross-link related topics
6. **Exercises**: Include hands-on activities with clear success criteria
7. **Assessment**: 5-10 questions per chapter for review

## üöÄ Running the Textbook Locally

### Prerequisites
- Node.js 18+ installed
- npm (comes with Node.js)

### Local Development

```bash
# Navigate to project directory
cd C:\Users\FC\Documents\HACKATHON\ai-book

# Install dependencies (if not already done)
npm install

# Start development server
npm start
```

The site will open at `http://localhost:3000/ai-book/`

### Building for Production

```bash
# Create production build
npm run build

# The static files will be in the `build/` directory

# Test production build locally
npm run serve
```

### Deployment to GitHub Pages

1. **Update Configuration**:
   - In `docusaurus.config.ts`, replace `YOUR_GITHUB_USERNAME` with your actual GitHub username (2 occurrences)

2. **Create GitHub Repository**:
   ```bash
   # Initialize git (if not already done)
   git init
   git add .
   git commit -m "Initial commit: Physical AI & Humanoid Robotics textbook"

   # Create repository on GitHub named "ai-book"
   # Then push:
   git remote add origin https://github.com/YOUR_USERNAME/ai-book.git
   git branch -M main
   git push -u origin main
   ```

3. **Deploy**:
   ```bash
   # Set up deployment credentials
   GIT_USER=YOUR_GITHUB_USERNAME npm run deploy
   ```

4. **Enable GitHub Pages**:
   - Go to repository Settings ‚Üí Pages
   - Source: Deploy from a branch
   - Branch: `gh-pages` / `root`
   - Save

5. **Access**: Your site will be live at `https://YOUR_USERNAME.github.io/ai-book/`
